Connecter Firebase ?

Cette application n'est pas encore connectée à Firebase. En la configurant, le planning et les mots de passe seront partagés automatiquement avec toute l'équipe. Sans ça, l'application fonctionne seule, sur cet appareil uniquement.

Configurer Firebase

Firebase (gratuit, par Google) va stocker le planning et les mots de passe pour toute l'équipe. Comptez environ 10 minutes, en 5 étapes courtes. Gardez cet écran ouvert dans un onglet, et ouvrez console.firebase.google.com dans un autre.

1 · Créer le projet

  1. Allez sur console.firebase.google.com, connectez-vous avec un compte Google.
  2. Cliquez « Créer un projet ».
  3. Donnez un nom (ex. le nom de votre équipe), Continuer.
  4. Désactivez Google Analytics, Créer le projet, patientez, puis Continuer.

2 · Créer la base de données

  1. Menu de gauche → Build → Realtime Database.
  2. Créer une base de données → emplacement Belgium (europe-west1) → Suivant.
  3. Choisissez « Démarrer en mode test » → Activer.

3 · Sécuriser (Règles)

  1. Dans Realtime Database, onglet « Règles » en haut.
  2. Effacez tout, collez ceci :
{
  "rules": {
    "config": {
      "teamName": { ".read": true },
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "planning": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "hidden": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "catIcons": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "catStyle": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    }
  }
}
  1. Cliquez Publier.

4 · Activer l'authentification

  1. Menu de gauche → Build → Authentication → Get started.
  2. Cliquez « E-mail/Mot de passe » dans la liste.
  3. Activez le premier interrupteur → Enregistrer.

5 · Récupérer la configuration

  1. Roue dentée ⚙️ (en haut à gauche) → Paramètres du projet.
  2. Section « Vos applications » → icône </> (Web).
  3. Surnom au choix, ne cochez pas Hosting, → Enregistrer l'application.
  4. Recopiez les 7 valeurs affichées ci-dessous :

Connexion Firebase

État : —

📖 Procédure complète pas à pas (à lire si vous débutez)

A. Créer le compte et le projet

  1. Ouvrez un navigateur et allez sur le site console.firebase.google.com
  2. Connectez-vous avec une adresse Google (Gmail). Si vous n'en avez pas, créez-en une d'abord.
  3. Cliquez sur le bouton « Créer un projet » (ou « Add project »).
  4. Tapez un nom, par exemple sparte-emasi, puis cliquez Continuer.
  5. Sur l'écran « Google Analytics », désactivez l'interrupteur (inutile ici), puis Continuer / Créer le projet.
  6. Patientez ~30 secondes, puis cliquez Continuer. Vous arrivez sur le tableau de bord du projet.

B. Créer la base de données

  1. Dans le menu de gauche, cliquez sur « Création » (ou « Build ») pour le dérouler.
  2. Cliquez sur « Realtime Database » (Base de données en temps réel). ⚠️ Pas « Firestore ».
  3. Cliquez sur le bouton « Créer une base de données ».
  4. Pour l'emplacement, choisissez « Belgium (europe-west1) », puis Suivant.
  5. Choisissez « Démarrer en mode test », puis cliquez Activer.
  6. Une page avec un tableau vide s'affiche : la base est créée. ✅

C. Autoriser l'accès (Règles sécurisées)

  1. En haut de cette même page, cliquez sur l'onglet « Règles » (Rules).
  2. Effacez tout le texte présent et collez exactement ceci :
{
  "rules": {
    "config": {
      "teamName": { ".read": true },
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "planning": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "hidden": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "catIcons": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    },
    "catStyle": {
      ".read": "auth != null",
      ".write": "auth != null && auth.token.email == 'admin-emasi-planning@compte.local'"
    }
  }
}
  1. Cliquez sur « Publier » (Publish). Confirmez si demandé.

🔒 Ces règles ferment tout à la base : seul un appareil connecté (via l'app) peut lire, et seul le compte administrateur peut écrire. Rien n'est accessible publiquement.

D. Activer l'authentification (obligatoire pour la sécurité)

  1. Dans le menu de gauche, cliquez sur « Build » → « Authentication ».
  2. Cliquez « Get started » (ou « Commencer »).
  3. Dans la liste des fournisseurs, cliquez sur « E-mail/Mot de passe ».
  4. Activez l'interrupteur du haut (« E-mail/Mot de passe »), laissez le second désactivé, puis Enregistrer.

Sans cette étape, l'ouverture de l'application et l'espace administrateur ne fonctionneront pas (erreur de connexion Firebase).

E. Récupérer la configuration

  1. Tout en haut à gauche, cliquez sur la roue dentée ⚙️ → « Paramètres du projet ».
  2. Descendez jusqu'à la section « Vos applications ».
  3. Cliquez sur l'icône </> (le symbole « Web »).
  4. Donnez un surnom (ex. EMASI), ne cochez pas « Hosting », puis Enregistrer l'application.
  5. Un bloc de code apparaît avec const firebaseConfig = {…}.
  6. Recopiez chaque valeur (entre les guillemets) dans les champs ci-dessous. Exemple : la ligne apiKey: "AIza…" → mettez AIza… dans le champ « apiKey ».
  7. Cliquez enfin sur « Enregistrer & connecter » en bas. L'app se recharge et se connecte.

💡 Astuce : laissez l'onglet Firebase ouvert pendant que vous recopiez les 7 valeurs. Vous pouvez copier-coller chaque valeur une par une.

🔒 Les règles sécurisées (auth requise, écriture réservée à l'admin) et l'activation de l'authentification sont détaillées dans la procédure ci-dessus (étapes C et D).
